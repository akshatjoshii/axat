UseContext - Maintain States through the Components Tree

How to use useContext to share specific data from all levels of application in React.

Akshat Â  Follow

# Introduction

Here we will cover how to use Context Api and react hook useContext in our react project. Context API is a specific structure though which we share specific data from all the levels of application in React.

# Declaration

React provides Context API as a way to pass data or state through the component tree without using props manually through each nested components. Context API uses ContextProvider and Context.

For example, lets assume we create a file auth-context.js with the following code.

import React from 'react';

// Creating context object and passing default data;
const AuthContext = React.createContext({
   isLoggedIn: null,
   login: () => {}
});
export default AuthContext;

# Adding Context to the Main Component

In order to use AuthContext in our main component, we need to import it by using import statement in our App.js file:

import React, { useState } from 'react';
import Auth from './Auth';
import AuthContext from './auth-context';

const App = () => {
    const [authStatus, setAuthStatus] = useState(false);
    const login = () => { 
     setAuthStatus(true); 
    };
    return (
        <>
           <AuthContext.Provider value={{ isLoggedIn: authStatus, login: login }}>
               <Auth />
           <AuthContext.Provider>
        </>
    );
};
export default App;

Note that, we have used .Provider component from AuthContext. .Provider enables the data in our context throughout our application. The data is passed in the value attribute of the .Provider component.

# useContext Hook

In .Provider component we passed the data, but how to get that data in children components? Here, in our case Auth component inside AuthContext.Provider.

Let's create Auth.js

import React, { useContext } from 'react';
import AuthContext from 'auth-context';

const Auth = () => {
    const auth = useContext(AuthContext);
    return (
    <div>
        <h1>Are you logged in?</h1>
        {auth.isLoggedIn? <div>Welcome User, You have successfully logged in!</div> : <div>Please login</div>}
        <button onClick={auth.login}>Login</button>        
    </div>
    )
} 
export default Auth;

We imported useContext hook, which is a named export provided by react and passed Authcontext as an argument to useContext and assigned it to a variable auth. Now auth contains the data which was passed from AuthContext.Provider. When we click at login button, login function in App component gets triggered and isLoggedIn becomes true. And the changes in the DOM renders accordingly.
